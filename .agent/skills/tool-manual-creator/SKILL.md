# 自作ツール説明書作成 Skill (Tool Manual Creator)

このスキルは、ユーザーが開発した自作のスクリプト、プラグイン、拡張機能、CLIツールなどの説明書（READMEやユーザーマニュアル）を高品質で生成するためのものです。

## 🎯 目的
- 開発したツールの魅力や使い方を、エンドユーザー（または未来の自分）に正確かつ分かりやすく伝える。
- 動作環境の確認からインストール、実際の使用方法、トラブルシューティングまでの導線を明確にする。
- プロフェッショナルな見た目と構成を持つ、Markdown形式およびPDF形式のドキュメントを提供する。
- マニュアルと連動した、魅力的で分かりやすいツールの紹介動画をあわせて作成する。
- 作成したツールを、WindowsおよびMacの両環境ですぐにインストールできるよう適切にパッケージ化（配布用データ作成）する。

## 📋 基本構造テンプレート
説明書を生成する際は、以下の構成をベースとし、対象のツールに合わせて柔軟に調整してください。

```markdown
# [ツール名]

[ツールの概要・キャッチコピー：1〜2文で「何ができるツールなのか」「どんな課題を解決するのか」を簡潔に説明します]

## ✨ 特徴 (Features)
- [特徴1：メインとなる最大のメリット]
- [特徴2：便利な機能]
- [特徴3：対応している形式など]

## 💻 動作環境 (Requirements)
- **対応OS**: [Windows 10/11 / macOS 13以降 など]
- **対応アプリケーション/バージョン**: [After Effects 2024以上 / Premiere Pro 2025 など]
- **必須コンポーネント**: [ZXPInstaller や 特定の拡張機能などが必要な場合]

## 📦 インストール方法 (Installation)
1. [ステップ1: ZIPファイルのダウンロードと解凍]
2. [ステップ2: 所定のディレクトリ（例: `~/Library/Preferences/Adobe/After Effects/...`）へファイルをコピー]
3. [ステップ3: アプリケーションを再起動し、メニューの `ウィンドウ > エクステンション` 等から実行]

## 🚀 使い方 (Usage)
1. [ステップ1: 対象となるレイヤーやアイテムを選択する]
2. [ステップ2: 本ツールのパネルから「〇〇」ボタンをクリックする]
3. [ステップ3: 処理が完了し、結果が反映される]

> **💡 Tips**: [知っておくと便利な使い方やショートカットなどの補足情報]

## ⚙️ 各UI・設定の説明 (Interface & Settings)
- **[Aボタン]**: [ボタンAの機能説明]
- **[Bスライダー]**: [スライダーBの機能説明]
- **[オプションチェックボックス]**: [ONにするとどうなるかを説明]

## ❓ よくある質問・トラブルシューティング (FAQ / Troubleshooting)
- **Q. [ボタンを押しても何も起きない / エラーが出る]**
  - A. [確認すべき項目（レイヤーを選択しているか、アクセス権限があるか等）を記載]
- **Q. [インストール先がわからない]**
  - A. [OSごとのインストール先パスの再掲など]

## 📜 変更履歴 (Changelog)
- **v1.0.0** (YYYY-MM-DD): 初版リリース

## 📄 ライセンス・免責事項 (License & Disclaimer)

- [再配布の可否や商用利用についてのルール等を記載]

## 👨‍💻 作者 (Author)
```

## 🤖 AIエージェント（あなた）への指示 (Instructions for Agent)

1. **対象の限定と情報収集**: ユーザーが明確に指定したツール（特定のスクリプト、プラグイン名称など）についてのみ説明書を作成してください。無関係なツールやフォルダ単位での全一括生成は行わないでください。ユーザーから「プラグインのコード」や「概要・機能案」を与えられたら、まずはそのツールが「何を行うものか」を分析してください。説明書を書く上で不足している情報（対応OS、具体的なパスなど）があれば、ユーザーに質問するか、プレースホルダーを設けて提案してください。
2. **フォーマットの適用**: 上記の基本構造テンプレートをベースに、ツールの性質に合わせてセクションを追加・削除してください。たとえば、スクリプト（.jsx）と拡張機能（CEP/UXP）ではインストール方法が全く異なるため、正確なパスを記載するよう留意してください。
3. **トーン＆マナー**: ユーザー（開発者）のユーザー（顧客）が見ることを想定し、丁寧で分かりやすく、かつ親しみやすい日本語を使用してください。（例：絵文字を適度に活用する、太字やリストを活用して視認性を高める）
4. **出力 (PDF)**: マニュアルは **PDF のみ** で作成する（`.md` ファイルは配布パッケージに含めない）。`Manual.pdf` としてユーザーのルールに従い `/Users/takahashikenta/projects/ソフト名/ツール名/` ディレクトリに保存します。PDF生成にはターミナルで `npx -y md-to-pdf` や `pandoc` を使用してください。
5. **紹介動画の生成**: マニュアルの作成後、その文面から「一言概要」「主な特徴」「使用ステップ」などを抽出して、ツールの紹介・プロモーション動画を制作してください。動画作成には、必要に応じて `remotion-video` プロジェクト内の既存スキルを利用し、動画ファイル（.mp4）も同じツールフォルダに出力してユーザーに提供してください。
6. **パッケージ化 (Packaging)**: マニュアルや動画の作成に加え、そのツール本体がWindows / Macどちらの環境でもユーザーがすぐにインストール・利用できるよう、適切な配布用パッケージ（例: 必要なリソースやフォルダを含めたZIP化、UXP/CEP拡張機能の場合は `.ccx` や `.zxp` のビルド生成など）を作成してください。
